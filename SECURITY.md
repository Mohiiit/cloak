# Security Policy

## Supported Versions
Security support currently applies to the `main` branch.

## Reporting a Vulnerability
Please do not report security vulnerabilities in public GitHub issues.

Send a private report to:
- Email: `mohitdhattarwal.personal@gmail.com`

Please include:
- A clear description of the issue
- Reproduction steps or proof of concept
- Impact assessment
- Suggested remediation (if available)

## Disclosure Process
- We will acknowledge receipt as soon as possible.
- We will investigate and validate the report.
- We may request additional details.
- Once fixed, we will coordinate disclosure timing with the reporter where possible.

## Scope Notes
Items most relevant for reports:
- Private key handling and storage
- Transaction signing and authorization flows
- Wallet/account state transitions
- API key or secret leakage in repository, CI, or build artifacts
